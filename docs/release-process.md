# Release Process

Use this workflow when preparing a GitHub release for `syft-cli-skills`.

## 1. Update Source Skills

Make all skill-content changes in `source-skills/`.
If behavior changes, update examples and installation docs as needed.

## 2. Rebuild Mirrors

Run:

```powershell
python .\scripts\build_skill_bundle.py
```

This refreshes:

- `.github/skills/`
- `codex-skills/`

## 3. Validate

Run:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\validate_skills.ps1
```

Do not publish a release if validation fails.

## 4. Create The Release Zip

Run:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\create_release_bundle.ps1 -Version v0.1.0
```

This creates a zip file under `release/`.

## 5. Push To GitHub

Create the public repository if needed, then push the repository contents.

## 6. Create The GitHub Release

Suggested release assets:

- source archive generated by GitHub
- the generated bundle zip from `release/`

Suggested release notes:

- new or changed skills
- installation-impacting changes
- example or documentation changes
- known limitations

## 7. Post-Release Checks

- confirm the README renders correctly
- confirm the mirrored skill directories are present in GitHub
- confirm the zip downloads and extracts cleanly
- confirm installation docs still match the current client behavior
